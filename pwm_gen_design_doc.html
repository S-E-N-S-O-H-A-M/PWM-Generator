1. Overview

A synthesisable Pulse Width Modulation (PWM) generator in Verilog with variable duty cycle. The duty cycle can be changed on-the-fly; a double-buffered duty register ensures glitch-free transitions at period boundaries.

Key Features

  • Parameterised resolution via PWM_BITS (default 8 → 256 duty levels).
  • Clock prescaler (PRESCALER) to lower the PWM frequency.
  • Double-buffered duty register — new value takes effect at the next period boundary.
  • Enable pin — output forced LOW when en = 0.
  • Fully synchronous with asynchronous active-low reset.

2. PWM Principle

A PWM signal is a square wave whose duty cycle (fraction of time spent HIGH) is varied to encode an analog-like value digitally.

PWM Frequency = clk / (PRESCALER × 2PWM_BITS)

Example: 50 MHz clock, PRESCALER=1, 8-bit → 50 000 000 / 256 ≈ 195.3 kHz

5. Parameters

ParameterDefaultDescription
PWM_BITS8 Counter width — gives 2N duty levels
PRESCALER1 Clock divider (1 = no division)
Duty cycle formula:
Duty % = duty_in / 2PWM_BITS × 100
Example (8-bit): duty_in = 128 → 128/256 × 100 = 50%

6. Double-Buffer Mechanism

To prevent glitches, duty_in is not used directly by the comparator. Instead it is latched into duty_buf only when the counter wraps from 2N−1 back to 0. This guarantees a clean, complete pulse every period regardless of when the user changes duty_in.

      Period N                   Period N+1
  ┌─────────────────┐       ┌─────────────────┐
  │ duty_buf = OLD   │       │ duty_buf = NEW   │
  └─────────────────┘       └─────────────────┘
                    ▲ counter wraps — latch duty_in → duty_buf
                    │
        duty_in changed here (mid-period) — no effect until wrap

7. RTL Source — pwm_gen.v

module pwm_gen #(
    parameter PWM_BITS  = 8,
    parameter PRESCALER = 1
)(
    input  wire                 clk,
    input  wire                 rst_n,
    input  wire                 en,
    input  wire [PWM_BITS-1:0]  duty_in,
    output reg                  pwm_out
);

    reg [PWM_BITS-1:0] counter;
    reg [PWM_BITS-1:0] duty_buf;
    reg [15:0]         pre_cnt;

    wire tick = (pre_cnt == PRESCALER - 1);

    // Prescaler
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)        pre_cnt <= 16'd0;
        else if (tick)     pre_cnt <= 16'd0;
        else               pre_cnt <= pre_cnt + 1'b1;
    end

    // PWM counter + duty buffer
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            counter  <= {PWM_BITS{1'b0}};
            duty_buf <= {PWM_BITS{1'b0}};
        end else if (tick) begin
            if (counter == {PWM_BITS{1'b1}}) begin
                counter  <= {PWM_BITS{1'b0}};
                duty_buf <= duty_in;       // latch at period boundary
            end else
                counter <= counter + 1'b1;
        end
    end

    // Comparator + enable gate
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)    pwm_out <= 1'b0;
        else if (!en)  pwm_out <= 1'b0;
        else           pwm_out <= (counter < duty_buf);
    end
endmodule

8. Testbench — pwm_gen_tb.v

`timescale 1ns / 1ps
module pwm_gen_tb;

    parameter PWM_BITS  = 8;
    parameter PRESCALER = 1;
    localparam PERIOD   = (1 << PWM_BITS);
    localparam HALF_CLK = 5;

    reg                  clk;
    reg                  rst_n;
    reg                  en;
    reg [PWM_BITS-1:0]   duty_in;
    wire                 pwm_out;

    pwm_gen #(.PWM_BITS(PWM_BITS), .PRESCALER(PRESCALER)) dut (
        .clk(clk), .rst_n(rst_n), .en(en),
        .duty_in(duty_in), .pwm_out(pwm_out)
    );

    initial clk = 0;
    always #(HALF_CLK) clk = ~clk;

    integer pass_count = 0, fail_count = 0;

    task automatic measure_duty;
        input [PWM_BITS-1:0] expected;
        integer high_cnt, i, diff;
        begin
            high_cnt = 0;
            repeat (PERIOD + 4) @(posedge clk);
            for (i = 0; i < PERIOD; i = i + 1) begin
                @(posedge clk);
                if (pwm_out) high_cnt = high_cnt + 1;
            end
            diff = high_cnt - expected;
            if (diff < 0) diff = -diff;
            if (diff <= 2) pass_count = pass_count + 1;
            else            fail_count = fail_count + 1;
        end
    endtask

    initial begin
        $dumpfile("pwm_gen_tb.vcd");
        $dumpvars(0, pwm_gen_tb);
        rst_n = 0; en = 0; duty_in = 0;
        repeat (10) @(posedge clk); rst_n = 1;
        repeat (5)  @(posedge clk);

        // Test: disabled, 0%, 25%, 50%, 75%, ~100%,
        //       mid-period change, disable, re-enable
        en = 1;
        duty_in = 8'd0;   measure_duty(8'd0);
        duty_in = 8'd64;  measure_duty(8'd64);
        duty_in = 8'd128; measure_duty(8'd128);
        duty_in = 8'd192; measure_duty(8'd192);
        duty_in = 8'd255; measure_duty(8'd255);

        $display("PASSED=%0d FAILED=%0d", pass_count, fail_count);
        $finish;
    end
endmodule

Checking Method

  • The measure_duty task counts HIGH samples over one full PWM period (256 ticks).
  • Result compared to duty_in with ±2 tick tolerance (accounts for pipeline latency).
  • Pass/fail counters printed at the end.

11. Synthesis Notes

Estimated FPGA resources (Xilinx 7-series, 8-bit, PRESCALER=1):
~20 LUTs, ~28 FFs. Fmax > 300 MHz.

Tip: For LED dimming, use PWM_BITS = 8 and PRESCALER = 195 with a 50 MHz clock to get ~1 kHz PWM — invisible flicker.

13. Possible Extensions

  • Dead-time insertion — for complementary outputs in H-bridge drivers.
  • Phase-correct (centre-aligned) PWM — counter counts up then down.
  • Multiple channels — parameterise number of independent outputs.
  • Register interface — APB/AXI-Lite for runtime configuration from a CPU.
  • Interrupt on period boundary — notify firmware to update duty.
